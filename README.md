# LXP Ноутбуки — учёт и выдача ноутбуков в колледже

MVP-модуль: преподаватель выдаёт студентам ноутбуки на пару, видит, кто что держит, и освобождает технику. Бэкенд хранит данные и правила, фронтенд показывает таблицу и форму выдачи.

## Стек

- **Frontend:** Vue 3 + Vite
- **Backend:** Python + Flask + SQLAlchemy + Gunicorn
- **БД:** PostgreSQL

## Структура

```text
LXP_PLUS/
├── Back/
│   ├── app/
│   │   ├── __init__.py     # сборка приложения (фабрика), БД, CORS
│   │   ├── config.py       # настройки, строка подключения к БД
│   │   ├── extensions.py   # объект db
│   │   ├── models.py       # таблицы: teachers, classrooms, laptops, issues, issue_items
│   │   └── routes.py       # все эндпоинты /api/*
│   ├── run.py              # запуск сервера
│   ├── seed.py             # создание таблиц и демо-данных
│   ├── requirements.txt
│   └── .env.example
└── Front/
    ├── src/ (App.vue, api.js, main.js, style.css)
    ├── index.html
    ├── package.json
    └── vitr.config.js
```

## Что нужно установить заранее

- **Python 3.10+** — проверка: `python --version` (на части систем `python3`)
- **PostgreSQL 13+** — должен быть установлен и запущен
- **Node.js 21+** и npm — проверка: `node --version`

## Запуск бэкенда

Сначала один раз создай базу и настрой подключение, затем подними сервер.

### Шаг 1. Создать базу в PostgreSQL

```sql
CREATE DATABASE lxplaptops_db;
```
(через `psql`, pgAdmin или любой клиент).

### Шаг 2. Настроить подключение

Скопируй `.env.example` в `.env` и впиши свои данные:
```
DATABASE_URL=postgresql://postgres:ПАРОЛЬ@localhost:5432/lxp_notebooks
```
- `postgres` — пользователь PostgreSQL, `ПАРОЛЬ` — его пароль;
- `localhost:5432` — адрес и порт базы;
- `lxp_notebooks` — имя базы из шага 1.

Если у пользователя нет пароля — убери `:ПАРОЛЬ`:
`postgresql://postgres@localhost:5432/lxp_notebooks`.

### Шаг 3. Установить зависимости и запустить

**macOS / Linux:**
```bash
cd Back
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python seed.py        # создаёт таблицы и демо-данные (34 ноутбука)
python run.py         # сервер на http://localhost:5000
```

**Windows (PowerShell или cmd):**
```bat
cd Back
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python seed.py
python run.py
```

Проверка, что бэкенд жив: открой `http://localhost:5000/api/health` — должно вернуть `{"status": "ok"}`.

> `python seed.py` пересоздаёт таблицы и стирает данные — запускай его только при первой настройке или когда нужно сбросить базу к демо-состоянию.

## Запуск фронтенда

Одинаково на всех ОС (нужен установленный Node.js). **Во втором окне терминала**, не закрывая бэкенд:

```bash
cd Front
npm install
npm run dev           # фронт на http://127.0.0.1:5173
```

Открой `http://127.0.0.1:5173` — это и есть приложение.

По умолчанию фронт обращается к бэкенду на `http://localhost:5000/api`. Если бэкенд на другом адресе, создай файл `Front/.env` со строкой:
```
VITE_API_BASE_URL=http://адрес-бэкенда:порт/api
```

## Как пользоваться

- **Карточки сверху** (Всего / Свободны / Заняты / Мои) — это ещё и фильтры: клик по карточке фильтрует таблицу.
- **«Взять ноутбуки»** открывает форму выдачи: выбираешь преподавателя, аудиторию, название пары и добавляешь от 1 до 10 ноутбуков, под каждый — ФИО ученика, группу, мышь/зарядку, число недостающих клавиш.
- Кнопка **«Очистить»** (✕) в строке освобождает один ноутбук вручную.
- Текущий пользователь зашит во фронте как «Юдин М. Д.» — его выдачи помечаются «ваше бронирование» и попадают в «Мои бронирования». (Когда появится настоящий вход, это место заменяется на реального пользователя.)

## Бизнес-правила (заложены в бэкенде)

- В одной выдаче от 1 до 10 ноутбуков.
- Один ноутбук нельзя выдать дважды в одной выдаче.
- Один ученик (ФИО + группа) не может держать два ноутбука одновременно.
- Выдать можно только свободный ноутбук.
- История выдач хранит срок `expires_at` (7 дней).

## API (эндпоинты)

| Метод  | Адрес                          | Что делает                          |
|--------|--------------------------------|-------------------------------------|
| GET    | /api/health                    | проверка «сервер жив»               |
| GET    | /api/teachers                  | список преподавателей               |
| GET    | /api/classrooms                | список аудиторий                    |
| GET    | /api/laptops                   | список ноутбуков с текущим статусом |
| POST   | /api/issues                    | создать выдачу (1-10 ноутбуков)     |
| POST   | /api/issues/&lt;id&gt;/return  | вернуть всю выдачу                  |
| POST   | /api/laptops/&lt;id&gt;/release| освободить один ноутбук             |
| GET    | /api/history                   | список выдач (история)              |
